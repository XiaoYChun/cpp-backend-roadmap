# Logger 类的设计

## 初步设计 VERSION_1
### 组件职责
Logger 是一个用于输出日志消息的组件。基本需求如下：
- Logger 需要支持两种输出形式（应该允许同时开启）——控制台输出和文件输出；
- Logger 的基本输出格式为：
  > [日志等级][日志时间] 日志信息。
- 其中 Logger 的日志等级，应该支持 INFO、WARN、ERROR 等多个等级，并且保留等级的拓展；
- Logger 的日志时间统一格式为 [YYYY-MM-DD-hh:mm:ss]；
- Logger 的日志信息由使用者提供；
- Logger 可以设置只输出特定等级及以上的日志信息。

### 使用场景
我们优先设想用户，考虑 Logger 的用户主要有哪些类型，主要是不同日志输出需求的用户，如需要输出系统阶段日志的程序段、用于 debug 的日志输出等。初步设想几个代表场景：
- 主线程在系统的不同阶段的开始、结束或是重要节点，使用 Logger 控制台输出日志，体现系统当前状态；
- 多个工作线程执行不同任务，并行的在同一个日志文件中记录各个任务的状态；
- 开发者正在 debug 某一段程序，调试过程中，Logger 可以只输出 DEBUG 等级的日志。

### 对外接口
- Log(LogLevel, LogContent)：用户通过 Log(...) 使用 Logger 完成日志的输出，具体输出位置、形式等根据当前的配置进行；

### 生命周期
生命周期关注资源的变化，所以主要根据核心资源的视角去看待：  
构造 Logger -> 打开输出文件 -> 持续输出 log() -> 析构关闭文件。
